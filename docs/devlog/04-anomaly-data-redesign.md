# 04 · 합성 이상 데이터 재설계 (탐지 가능성)

분석 레이어를 만들기 전에, 주입한 이상이 실제로 탐지 가능한지 측정 → 결함 발견 →
데이터 모델을 재설계했다.

## 배경: 측정으로 드러난 결함

분석 레이어가 의미 있으려면 데이터에 탐지 가능한 신호가 있어야 한다. 기존 Day 1
데이터를 측정하니:
- **조명 이상이 깨짐**: 이상 야간 28.2 < 정상 야간 30.1 (탐지 불가)
- 압축기 이상도 약함 (정상 범위와 겹침)

원인: 설비 base load가 항상 켜져 있어 idle 상태가 분리되지 않음 → "야간에 켜짐"이
정상과 구분 안 됨.

## 재설계: idle + active 모델

- 각 설비 = **idle(상시 base) + active(평일 근무시간만)**.
- 야간·주말엔 active=0 → idle로 낮게 떨어짐 → "쉬어야 할 때 돌아감"이 명확한 이상.
- `scripts/verify_anomalies.py`: 각 이상을 **매칭 baseline**(같은 시간대/요일유형)과
  비교해 비율로 탐지 가능성 검증 (≥2배 = PASS). "데이터에 신호 있음"을 보증.

## 적대적 리뷰와 수정

리뷰가 HVAC 이상의 결함 2건을 잡음:
- **MAJOR**: HVAC 스파이크가 `2026-05-09`(토요일)에 주입됨 → is_workday=0이라 정상이
  idle(20)이지 "근무 peak(140)"이 아님. 주석이 거짓이고 의도한 내러티브와 다름.
  → 평일(`2026-05-08` 금) 오후로 이동, 320kW.
- **MAJOR**: 검증기 HVAC baseline `~hvac_spike`가 월 전체 평균(46.7)으로 희석 → 약한
  결함을 PASS로 오판 위험. → **매칭 baseline**(평일 13-16, 스파이크 제외)으로 수정.
- 부가: NaN 가드, CSV 경로를 스크립트 기준 절대경로로.

## 결과

| 이상 | anomaly | baseline | 비율 | 판정 |
|---|---|---|---|---|
| 압축기 주말 고착 | 150.0 | 29.2 | 5.1× | PASS |
| 조명 야간 방치 | 30.0 | 3.1 | 9.7× | PASS |
| HVAC 스파이크 | 320.0 | 110.4 | 2.9× | PASS |

세 이상 모두 정상을 크게 벗어남 → 모델 비교가 공정하고 유의미해짐. 각 이상은 RAG
가이드라인(압축공기/조명/공조)과 연결되어, 모델의 탐지+설명+근거 인용을 평가할 수 있다.
